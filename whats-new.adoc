=== Cloud Volumes ONTAP



==== 能夠管理 Cloud Volumes ONTAP 的 Console 代理程式 Proxy 設定

您現在可以在 NetApp Console agent 上管理 Cloud Volumes ONTAP 的代理伺服器設定，即使您失去連線或代理伺服器設定錯誤也沒問題。先前，如果 Console agent 無法在 20 分鐘內連接到 Cloud Volumes ONTAP，則會將您的手動代理伺服器設定覆蓋為預設設定。這會導致通訊失敗，包括 AutoSupport 訊息的問題。若要保留現有系統的代理伺服器設定，請執行以下 API 呼叫：

`PUT /occm/config`

請在請求內文中包含以下參數：

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
預設模式為標準模式，這表示如果 Console 代理程式在 20 分鐘內無法連線至 Cloud Volumes ONTAP，則會將您的 Proxy 設定覆寫為預設值。

https://docs.netapp.com/us-en/console-automation/cvo/put-occm-config.html["設定可修改的 NetApp Console 參數"^]

=== 備份和復原



==== ONTAP Volumes 工作負載增強功能

在此版本中、 ONTAP Volumes 工作負載引入了以下增強功能：

* *還原到來源位置*：現在您可以將本機快照和複寫快照中的檔案還原到來源位置以及備用位置。
* *每小時磁碟區探索*：Dashboard 現在可以更快地探索和重新整理新磁碟區。
* *支援暗網站*：您現在可以從 https://mysupport.netapp.com/site/products/all/details/netapp-console/downloads-tab/download/61906/4.4.0/downloads["NetApp 支援網站"]下載適用於暗網站的 Backup and Recovery。目前，適用於暗網站的 NetApp Backup and Recovery 僅支援 ONTAP Volumes 工作負載。
* *Unicode 相容性改進*：目錄索引現在可以更好地處理 Unicode 字元。
* *改進的 FlexGroup 相容性*：將 ONTAP FlexVol 磁碟區轉換為 FlexGroup 磁碟區時、現在可以還原轉換之前建立的備份。
* *多重儲存桶支援*：現在您可以跨不同的雲端供應商，使用每個系統最多 6 個儲存桶來保護系統內的磁碟區。


有關保護 ONTAP Volume 工作負載的詳細資訊，請參閱 https://docs.netapp.com/us-en/data-services-backup-recovery/prev-ontap-protect-overview.html["保護 ONTAP Volume 工作負載概述"]。



==== VMware 工作負載增強功能

在此版本中、 VMware 工作負載引進下列增強功能：

* *從物件儲存還原客體資料*：您現在可以從物件儲存還原客體檔案和資料夾。
* *VMFS/NFS 快速還原*：現在，您可以在還原到備用位置時啟用快速還原選項。啟用後，系統只會從物件儲存中擷取 VM 中繼資料（檔案、LUN 和命名空間），使磁碟區的可用速度比完整還原更快。
* *支援 Amazon FSx for NetApp ONTAP*：VMware 工作負載現在支援執行 9.13.1 或更新版本的 Amazon FSx for NetApp ONTAP 儲存系統。


有關保護 VMware 工作負載的詳細資訊，請參閱 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-protect-overview.html["保護 VMware 工作負載概述"]。



=== 資料分類



==== 版本 1.52

.資源驗證
為確保可靠性，Data Classification 現在會在掃描、報告產生和執行操作過程中驗證可用記憶體和磁碟資源。當記憶體使用率超過 95% 或磁碟空間低於 6 GB 時，Data Classification 會自動暫停操作。資源充足後，操作將恢復。

若要監控您的系統資源、請參閱 link:https://docs.netapp.com/us-en/data-services-data-classification/task-health-monitor.html["監控 Data Classification 的運作狀況"^]。

.提高了複製和移動的限制
現在支援對 CIFS 目標進行複製和移動操作。這兩個操作的單一檔案大小限制已從先前的 50 MB 提高到 1 GB。這兩個操作的操作限制也已提高到 1 TB 或 500 萬個檔案。

如需詳細資訊，請參閱 link:https://docs.netapp.com/us-en/data-services-data-classification/task-copy-move.html["複製、移動和刪除資料"^]。

.Active Directory 整合設定權限
現在，資料分類中對 Active Directory（AD）整合設定的變更僅限於管理員。檢視者不再能夠修改 AD 設定。

有關 Classification 檢視器角色的更多資訊，請參閱 link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html#data-service-roles["資料服務角色"^]。

.支援 RHEL 9.7
資料分類功能新增了 Red Hat Enterprise Linux v9.7，可用於手動內部部署安裝，包括 dark site 部署。

以下作業系統需要使用 Podman 容器引擎，並且需要 Data Classification 版本 1.30 或更高版本：Red Hat Enterprise Linux 版本 8.8、8.10、9.0、9.1、9.2、9.3、9.4、9.5、9.6 和 9.7。

.透明 Proxy 支援
Data Classification 現在支援透明 Proxy 配置，從而能夠在 Proxy 或受限網路環境中更順利地部署。

如需在受限環境中部署的詳細資訊，請參閱 link:https://docs.netapp.com/us-en/data-services-data-classification/task-deploy-compliance-dark-site.html["在沒有網路連線的 Linux 主機上安裝 NetApp Data Classification"^]。

=== Cloud Volumes ONTAP



==== Cloud Volumes ONTAP 支援下一代 Google Cloud VM

在 9.18.1 版本中，NetApp 將新的 Cloud Volumes ONTAP 部署從 N2 虛擬機遷移到新一代 Google Cloud C3 系列虛擬機，帶來更快、更具可擴展性的體驗。現在，您可以在 Google Cloud 中部署 Cloud Volumes ONTAP 9.18.1 及更高版本時充分利用 C3 系列虛擬機器的優勢。C3 系列虛擬機器採用 Google Virtual NIC（gVNIC）和 Hyperdisk Balanced 磁碟，可確保為高強度工作負載提供動態效能，從而提供更高的效能和更大的容量限制。


NOTE: 目前，Cloud Volumes ONTAP 僅支援單節點部署中的 C3 系列。

如果您的 Cloud Volumes ONTAP 系統執行 9.18.1 或更新版本，用於輕鬆進行單節點部署的預先設定套件會自動使用 C3 VM，同時讓您能夠根據工作負載需求自訂 IOPS 和處理量參數。同樣地，在建立 Aggregate 時，您可以新增 Hyperdisk Balanced 磁碟，以在 Google Cloud 中實現更好的效能和擴充性。此外，您可以選擇 C3 系列機器的 LSSD 變體，以獲得預設的 Flash Cache 支援。

在向 Aggregate 中新增 Volume 時，無法變更 C3 VM 的磁碟類型，因為 C3 僅支援 Hyperdisk Balanced 磁碟。同樣，將 N2 VM 類型的系統複製到 C3 VM 時，磁碟類型預設為 Hyperdisk Balanced。

==== Azure 中 Cloud Volumes ONTAP 的 VNet 安全性

Cloud Volumes ONTAP 9.18.1 及更高版本在 Azure 單可用區和多可用區中的部署支援 Azure 虛擬網路（VNet）加密，作為其分層安全性策略的一部分，用於保護傳輸中的資料。Cloud Volumes ONTAP 利用 Azure 原生資料封包傳輸層安全性（DTLS）協定來保護 ONTAP 節點、管理介面和其他 Azure 服務之間的通訊，防止攔截和未經授權的存取。這種網路級加密與 ONTAP 內建的儲存和靜態資料保護機制相輔相成，為您的資料提供端對端的安全保障。
