=== Cloud Volumes ONTAP



==== 支援 Google Cloud Infrastructure Manager

Cloud Volumes ONTAP 9.16.1 及更高版本現在支援使用 https://docs.cloud.google.com/infrastructure-manager/docs/overview["Google Cloud Infrastructure Manager"^]（IM）而非 https://docs.cloud.google.com/deployment-manager/docs/deprecations["雲端部署管理器"^]（DM）來部署 Google Cloud 中的新部署。Google 將在不久的將來棄用 Deployment Manager 作為基礎架構服務，轉而使用更進階的基礎架構管理器。

現在，自動部署 Cloud Volumes ONTAP 以使用 Infrastructure Manager，或執行轉換工具將 Deployment Manager 中的現有部署切換到 Infrastructure Manager。轉換是一次性的過程，之後您的系統將自動開始使用 Infrastructure Manager。有關執行轉換工具的說明，請參閱 https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/task-gcp-dm-im-conv.html["為 Google Cloud Infrastructure Manager 設定現有的 Cloud Volumes ONTAP 部署"^]。

使用 Infrastructure Manager 的 Cloud Volumes ONTAP 系統會使用 Google Cloud Storage 儲存桶來儲存資料和記錄，這些儲存桶位於首次部署的區域中，用於儲存部署記錄，這些記錄可供後續部署重複使用。這些儲存桶可能會產生額外費用，但請勿編輯或刪除儲存桶及其內容：

* `gs://netapp-cvo-infrastructure-manager-<project id>/dm-to-im-convert`：用於儲存 Cloud Volumes ONTAP Terraform 文件
* `<gcp project number>-<region>-blueprint-config`：用於儲存 Google Cloud Terraform 工件


.相關連結
* https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/task-getting-started-gcp.html["開始在 Google Cloud 中使用 Cloud Volumes ONTAP"^]
* https://docs.netapp.com/us-en/console-setup-admin/whats-new.html#console-agent-4-2-0["NetApp Console Agent 4.2.0 發行說明"^]
* https://docs.netapp.com/us-en/console-setup-admin/reference-permissions-gcp.html#gcp-permissions-change-log["Google Cloud Infrastructure Manager 所需的權限"^]




== 2026 年 2 月 04 日



=== 數位顧問



==== 全面安全監督的新安全報告

Digital Advisor 現已包含安全報告功能，可提供整個儲存環境中 ONTAP 安全風險的統一視圖。該報告能夠更快地偵測叢集、SVM 和磁碟區中的配置錯誤、合規性漏洞和安全性問題，從而幫助您維護安全的儲存基礎架構。



==== 增強了對混合修補程式層級的 Upgrade Advisor 支援

Upgrade Advisor 現在可以為叢集中節點在相同 ONTAP 主要版本內執行不同修補程式層級產生升級報告。這項增強功能確保只要叢集未跨主要版本混用，客戶就可以無縫地進行升級，從而為升級規劃提供更大的靈活性。



==== 擴充支援已結束有限支援的 ONTAP 版本

Upgrade Advisor 現已支援為已結束有限支援的 ONTAP 版本產生升級計劃，並提供 9 個月的寬限期。此功能可在過渡期內提供清晰的指導，幫助客戶及時採取行動，避免升級中斷。

=== Keystone

NetApp Console中Keystone儀表板的新增強功能：



==== 每日累積突發資料的互動式篩選

「消費趨勢」標籤中的「按天計算累積峰值」表格現已支援互動式篩選。選擇累計峰值計費圖表中的任一長條圖，即可僅查看該計費週期內的天數。要了解更多信息，請參閱link:./integrations/consumption-tab.html#view-daily-accrued-burst-data-usage["檢視每日累計突發資料使用量"]。

*消費趨勢* 標籤中的累積突發帳單圖表現在顯示每個帳單週期的完整日期範圍，例如 Oct 01, 2025 to Oct 31, 2025，而不僅僅是月份和年份。

*訂閱*標籤包含一個選項，可以展開所有訂閱並同時查看每個訂閱的效能服務等級資訊。要了解更多信息，請參閱link:./integrations/subscriptions-tab.html["查看您的 Keystone 訂閱詳情"]。

*支援多個儲存桶*：（私有預覽）從ONTAP 9.17.1 及更高版本開始，您現在可以保護系統中的捲，每個系統最多可跨不同的雲端供應商使用 6 個儲存桶。

* 現在您可以還原客戶作業系統檔案和資料夾。


https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-restore-guest-files-and-folders-overview.html["了解更多關於恢復訪客文件和資料夾的信息"]。

NOTE: 僅在此版本中，您無法使用「操作」功能表中的「升級」選項升級 Hyper-V 或 Windows 的NetApp程式。相反，請刪除每個 Hyper-V 主機，然後重新新增以更新插件。

https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-hyperv-restore.html["了解更多關於使用NetApp Backup and Recovery恢復 Hyper-V 虛擬機器的信息"]。

=== 資料分類



==== 版本 1.50

本次資料分類版本更新包含錯誤修復和以下更新：

.自訂分類改進
資料分類功能現在支援建立自訂資料類別。您可以上傳檔案來微調資料分類所使用的 AI 模型，以便將類別標記套用至資料。所有自訂分類的介面都得到了改進。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-data-classification/task-custom-classification.html["建立自訂分類"^]。

.自訂過期資料定義
資料分類功能現在可讓您自訂過期資料的定義，以滿足您組織的需求。此前，過時數據被定義為三年前最後一次修改的數據。現在，可以根據上次訪問時間或上次修改時間來識別過時的數據；時間段可以從 6 個月前到 10 年前不等。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-data-classification/task-stale-data.html["自訂過期資料定義"^]。

.性能提升
資料分類、資料對應報告和調查頁面上的篩選器等所有頁面的載入時間都已縮短。

.調查報告預計完成時間
下載調查報告時，「資料分類」現在會顯示下載完成的預計時間。

=== 控制台設定和管理



==== 控制台代理 4.3.0

4.3.0 版本同時支援標準模式和受限模式。

此版本的控制台代理程式包含安全性改進、錯誤修復以及以下功能：

.能夠驗證控制台代理的網路連接
現在，您可以直接從 NetApp Console 驗證已連接 Console 代理的網路連線。此功能有助於驗證連接性並排查 Console 代理的問題。此外，您也可以透過 Console 代理程式維護主控台命令列介面（CLI）來執行網路診斷程式。

link:https://docs.netapp.com/us-en/console-setup-admin/task-troubleshoot-agent.html#verify-network-access["了解如何從 NetApp Console 運行配置。"]



==== NetApp Console管理

本次發布包含以下內容：

.NetApp Console中聯合群組的角色存取控制
NetApp Console支援將角色指派給聯合群組（例如 Active Directory 群組），從而允許管理員根據組織身分提供者 (IdP) 中的群組成員身分自動執行使用者入職和離職。此功能可減少管理開銷，並透過在群組成員身分變更時立即更新控制台存取權限來確保安全、一致的存取。

link:https://docs.netapp.com/us-en/console-setup-admin/task-iam-manage-members-permissions.html#add-a-federated-group-to-your-organization["了解如何為您的組織提供聯合群組的存取權限。"]

.NetApp Console處於受限模式時支援聯合身份驗證
現在您可以為受限模式的NetApp Console組織啟用聯合驗證。這樣，使用者就可以使用其公司憑證登入控制台，同時保持受限模式的安全優勢。

.唯讀模式
您可以將NetApp Console組織設定為唯讀模式。在唯讀模式下，使用者可以查看資源和設置，但不能進行任何更改。組織管理員或超級管理員可以啟用組織的唯讀模式。啟用唯讀模式後，具有管理角色的使用者必須手動提升權限才能根據需要進行變更。

link:https://docs.netapp.com/us-en/console-setup-admin/task-enable-read-only-mode.html["了解如何為控制台組織啟用唯讀模式。"]

link:https://docs.netapp.com/us-en/console-setup-admin/task-user-settings.html["當你的組織處於唯讀模式時，了解如何提升你的角色。"]



=== 災難復原



==== 支援本地環境中的多個控制台代理

如果您在本機使用災難恢復，現在可以為每個 vCenter 執行個體部署控制台代理，從而提高復原能力。

例如，如果您有兩個站點（站點 A 和 B），則站點 A 可以將控制台代理 A 連接到 vCenter 1、 ONTAP部署 1 和ONTAP部署 2。站點 B 可以將控制台代理 B 連接到 vCenter 2 和ONTAP部署 3 和 4。

有關災難復原中的控制台代理的信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/get-started/dr-prerequisites.html#netapp-console-prerequisites["建立控制台代理"^]。



==== 使用基於資料儲存的保護機制，在故障轉移後為複製計劃添加虛擬機

當故障轉移被觸發時，任何使用基於資料儲存的保護的複製計劃都會包含已新增至資料儲存的虛擬機，前提是這些虛擬機已被發現。故障轉移完成前，您必須提供新增虛擬機器的對應詳細資訊。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/failover.html["故障轉移應用程式"^]。



==== 新郵件通知

災難復原功能現在會針對以下事件發送電子郵件通知：

* 接近容量使用極限
* 報告產生完成
* 工作失敗
* 許可證到期或違規




==== Swagger 改進

現在您可以從災難復原中存取 Swagger 文件。在災難復原中，選擇**設定**，然後選擇**API 文件**以連結到 Swagger，或在瀏覽器的隱身/隱私模式下存取此 URL： link:https://snapcenter.cloudmanager.cloud.netapp.com/api/api-doc/draas["https://snapcenter.cloudmanager.cloud.netapp.com/api/api-doc/draas"^]。



==== 改進的使用者介面

災難復原功能現在提供更完善的警告和錯誤解決方法。此版本修正了導致已取消的作業無法在使用者介面中顯示的錯誤。已取消的職位現在可以查看了。當同一個目標網路映射到多個不同的來源網路時，也會發出新的警告。



==== 保留複製計劃中預設新增的 VM 資料夾結構

建立複製時，新的預設設定是保留 VM 資料夾結構。如果復原目標沒有原始資料夾層次結構，災難復原程式會建立它。您可以取消選擇此選項以忽略原始資料夾層次結構。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/drplan-create.html["建立複製計劃"^]。



=== 勒索軟體抵禦能力



==== 將快照複製到ONTAP

勒索軟體復原功能現在支援將快照複製到輔助ONTAP網站。透過使用複製策略的保護群組，您可以為每個工作負載複製到相同的目標位置或不同的目標位置。您可以建立包含複製功能的勒索軟體防護策略，也可以使用預先定義的策略。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-use-protect.html["在勒索軟體復原中保護工作負載"^]。



==== 將工作負載排除在勒索軟體復原能力之外

勒索軟體復原功能現在支援將系統中的特定工作負載從保護範圍和勒索軟體復原儀表板中排除。發現工作負載後，您可以將其排除在外；如果您想要添加勒索軟體防護，則可以重新將其包含在內。排除在外的工作負載無需付費。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-start-discover.html#exclude-workloads["排除工作負載"^]。



==== 標記提醒，如正在審核中

勒索軟體復原功能現在允許您將警報標記為「審核中」。使用「審核中」標籤可以提高團隊在對活躍的勒索軟體威脅進行分類和管理時的清晰度。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/rp-use-alert.html["管理勒索軟體復原能力中的警報"^]。

== 2025年12月10日



=== 控制台設定和管理



==== 控制台代理 4.2.0

4.2.0 版本同時支援標準模式和受限模式。

此版本的控制台代理程式包含安全性改進、錯誤修復以及以下功能：

.支援 Google Cloud Infrastructure Manager
NetApp現在使用 Google Cloud Infrastructure Manager (IM) 而不是 Google Cloud Deployment Manager 來部署代理程式和管理 Google Cloud 中的代理程式。做出此項更改是因為谷歌將棄用此功能。 link:https://docs.cloud.google.com/deployment-manager/docs/deprecations["雲端部署管理器"]。

* 任何 4.2.0 及更高版本的新代理程式都使用基礎架構管理器，您應該更新用於部署的使用者帳戶和服務帳戶權限。 link:https://docs.netapp.com/us-en/console-setup-admin/reference-permissions-gcp.html#gcp-permissions-change-log["查看權限變更日誌。"^]
* 部署代理程式時，系統也會建立一個 Google Cloud 儲存桶來儲存部署檔案。


.改進了控制台代理的配置檢查
* 控制台代理現在在執行設定檢查時會檢查已棄用的端點。如果您尚未更新至 4.0.0 或更高版本的新端點列表，則只要係統能夠存取先前的端點列表，安裝就會成功。 link:https://docs.netapp.com/us-en/console-setup-admin/reference-networking-saas-console-previous.html#update-endpoint-list["了解更多關於控制台代理程式所需端點的資訊。"^]
* 從控制台或代理程式維護控制台對已安裝的控制台代理程式執行設定檢查，以驗證連線性並排除故障。 link:https://docs.netapp.com/us-en/console-setup-admin/task-troubleshoot-agent.html#verify-network-access["了解如何對控制台代理程式運行配置檢查。"^]


.直接從NetApp Console下載代理軟體
