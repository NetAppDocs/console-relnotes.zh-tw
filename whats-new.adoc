== 2025年12月10日



=== 控制台設定和管理



==== 控制台代理 4.2.0

4.2.0 版本同時支援標準模式和受限模式。

此版本的控制台代理程式包含安全性改進、錯誤修復以及以下功能：

.支援 Google Cloud Infrastructure Manager
NetApp現在使用 Google Cloud Infrastructure Manager (IM) 而不是 Google Cloud Deployment Manager 來部署代理程式和管理 Google Cloud 中的代理程式。做出此項更改是因為谷歌將棄用此功能。 link:https://docs.cloud.google.com/deployment-manager/docs/deprecations["雲端部署管理器"]。

* 任何 4.2.0 及更高版本的新代理程式都使用基礎架構管理器，您應該更新用於部署的使用者帳戶和服務帳戶權限。 link:https://docs.netapp.com/us-en/console-setup-admin/reference-permissions-gcp.html#gcp-permissions-change-log["查看權限變更日誌。"^]
* 部署代理程式時，系統也會建立一個 Google Cloud 儲存桶來儲存部署檔案。


.改進了控制台代理的配置檢查
* 控制台代理現在在執行設定檢查時會檢查已棄用的端點。如果您尚未更新至 4.0.0 或更高版本的新端點列表，則只要係統能夠存取先前的端點列表，安裝就會成功。 link:https://docs.netapp.com/us-en/console-setup-admin/reference-networking-saas-console-previous.html#update-endpoint-list["了解更多關於控制台代理程式所需端點的資訊。"^]
* 從控制台或代理程式維護控制台對已安裝的控制台代理程式執行設定檢查，以驗證連線性並排除故障。 link:https://docs.netapp.com/us-en/console-setup-admin/task-troubleshoot-agent.html#verify-network-access["了解如何對控制台代理程式運行配置檢查。"^]


.直接從NetApp Console下載代理軟體
當您需要手動安裝代理程式時，除了NetApp支援網站之外，您還可以直接從NetApp Console存取代理程式軟體。 link:https://docs.netapp.com/us-en/console-setup-admin/task-install-agent-on-prem.html#install-the-console-agent-manually["了解如何直接從NetApp Console下載控制台代理軟體。"^]



==== NetApp Console管理

本次發布包含以下內容：

.能夠設定憑證過期通知。
設定服務帳戶和聯合身分驗證憑證過期的通知。選擇七天或三十天。控制台會向具有相應角色的使用者顯示通知和電子郵件。組織管理員會收到服務帳戶通知。組織管理員、聯盟管理員和聯盟檢視者會收到聯盟通知。

.啟用聯合身份驗證後，本地登入將不可用。
為控制台組織啟用聯合身份驗證後，使用者將無法使用本機登錄，而是會被引導至聯合身份驗證登入。

.提升儲存管理頁面的易用性
現在，您可以從儲存管理頁面更輕鬆地查看和管理有關ONTAP本機系統（以及 FSx for ONTAP）的詳細資訊。

* 「Discoverable systems」頁面將摘要資訊與可用系統的標籤顯示分開，從而更容易查看有關可發現系統的全面資訊。




=== 本地ONTAP集群



==== 從NetApp Console查看ONTAP系統時的可用性增強（僅限標準模式）

NetApp Console中的「系統」頁面在管理磁碟區和叢集時提高了清晰度和易用性。這些更新使用戶體驗與較新版本的ONTAP系統管理器更加一致，並簡化了導覽。

.ONTAP 9.5 及更高版本包含叢集容量（僅限控制台標準模式）
*系統 > 概述* 頁面會顯示叢集大小以及執行ONTAP 9.5 及更高版本的叢集的ONTAP版本。

對於受限模式下的NetApp Console或ONTAP版本 9.5 及更低版本，已設定磁碟區的資訊保持不變。

.如果您擁有超過 20 個本機ONTAP集群，則「系統」頁面上的「磁碟區」標籤預設顯示為網格視圖。
當您在NetApp Console中管理超過 20 個本機ONTAP叢集時，「磁碟區」標籤預設顯示網格視圖，以便於導覽。卡片視圖已停用。

此變更僅適用於標準模式，並且適用於ONTAP版本 9.6 及更高版本。



== 2025年12月9日



=== 災難復原



==== 資料夾層級結構保留

預設情況下，災難復原會在故障轉移時保留虛擬機器清單層次結構（資料夾結構）。如果復原目標中沒有所需的資料夾，災難復原功能會建立該資料夾。

現在您可以透過指定新的父 VM 資料夾或取消選取「保留原始資料夾層次結構」選項來覆寫此設定。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/drplan-create.html["建立複製計劃"]。



==== 簡化控制台代理更新

災難復原現在支援在工作環境中使用多個控制台代理的簡化流程。若要在控制台代理程式之間切換，您必須編輯 vCenter 配置，重新發現憑證，並重新整理複製計畫以使用新的控制台代理程式。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/reference/switch-console-agent.html["切換控制台代理"]。

=== Cloud Volumes ONTAP



==== 提升 Azure 中 Premium SSD v2 磁碟效能的能力

現在，您可以透過修改 IOPS 和吞吐量參數來提高 Azure 中 Premium SSD v2 託管磁碟的效能。利用此功能，您可以根據工作負載需求最佳化系統的儲存效能。

https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-manage-perf-params-azure.html["在 Azure 中管理Cloud Volumes ONTAP的 Premium SSD v2 磁碟效能"^]。



==== Essentials 授權超額收費簡化

對於Cloud Volumes ONTAP市場年度合約/私有報價，Essentials 授權的超額使用運算現在與自帶授權 (BYOL) 套餐保持一致。此前，超出部分按基本套餐的每小時市場價格計費。現在，如果您的市場年度合約包含多個 Essentials 套餐， NetApp Console會將 Essentials 套餐的超額費用計入您訂閱中價格較高的 Essentials 套餐的可用容量。這簡化了 Essentials 套餐的超額費用計算，並確保從 BYOL 授權模式平穩過渡到訂閱模式。

https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/concept-licensing-charging.html#how-overages-are-charged-for-essentials-licenses["Essentials許可證超額費用如何收取"^]



==== 支援 Azure Edsv6 尺寸系列

從Cloud Volumes ONTAP 9.17.1 開始，您可以透過NetApp Console為新的Cloud Volumes ONTAP執行個體部署 Azure Edsv6 系列虛擬機器。Cloud Volumes ONTAP 9.17.1 及更高版本將僅支援新部署的第二代虛擬機器。這些第二代機器與最新技術相容，例如統一可擴充韌體介面 (UEFI)、Azure Boost 系統和 NVMe。它們非常適合記憶體密集系統和需要快速本地儲存的應用，例如資料庫伺服器和分析引擎。

https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-azure.html#supported-configurations-by-license["Azure 中Cloud Volumes ONTAP支援的配置"^]

=== 災難復原



==== 級聯故障轉移支持

現在您可以在ONTAP中設定級聯關係，並使用該複製關係的任何分支進行災難復原。



==== 註冊期間降低 VMware 硬體支援級別

災難復原功能現在支援在註冊過程中將 VMware 硬體降級到早期版本的 vSphere。當來源 ESX 主機運行的版本比災難復原網站的版本更新時，這非常有用。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/drplan-create.html#select-vcenter-servers["在NetApp Disaster Recovery中建立複製計劃"]。



==== 優雅關機

災難復原現在會優雅地關閉虛擬機，而不是直接斷電。如果某個虛擬關機時間超過十分鐘，災難復原功能會將其關閉。



==== 備份前腳本支持

現在您可以將自訂腳本注入到故障轉移工作流程中，以便在建立備份之前執行。備份前腳本可讓您在複製快照之前控制虛擬機器的狀態，並為虛擬機器的轉換做好準備。例如，您可以注入腳本來卸載 NFS 掛載點，然後在故障轉移後使用另一個腳本重新掛載該掛載點。

有關詳細信息，請參閱 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/drplan-create.html#select-vcenter-servers["在NetApp Disaster Recovery中建立複製計劃"]。



=== 勒索軟體抵禦能力

此版本包含一般增強與改進。



=== Cloud Volumes ONTAP



==== 增強的 NVMe-TCP 支持

先前，在 NVMe-TCP 上部署Cloud Volumes ONTAP實例時，您必須在部署之前手動取得和套用 NVMe 授權。透過此更新， Cloud Volumes ONTAP現在會在部署期間自動安裝所需的 NVMe 許可證，從而簡化設定流程。

對於缺少許可證的現有 NVMe-TCP 部署， Cloud Volumes ONTAP會自動套用授權。您必須重新啟動系統才能使許可證生效。

更多資訊請參見 https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/concept-client-protocols.html#nvme-tcp["Cloud Volumes ONTAP支援的客戶端協定：NVMe-TCP"^]。



=== Keystone

NetApp Console中Keystone儀表板的新增強功能：



==== 填寫您的保險覆蓋範圍通知框

== 2025年10月13日



=== Keystone

NetApp Console中Keystone儀表板的新增強功能：



==== 資產標籤中的 QoS 策略覆蓋列

*Assets* 選項卡中的 *Volumes in clusters* 選項卡現在包含一個 *QoS policy override* 列，指示是否繞過備份和鏡像卷的 QoS 規則，顯示 *true*（忽略規則）、*false*（強制執行規則）或 *N/A*（對於鏡像卷）。


NOTE: 對於Keystone v3 訂閱，此列顯示 *N/A*，因為 QoS 策略不適用於 v3 產品。



==== 增強了概覽頁面中的警報可見性

*概覽*頁面現在包括兩個新的警報部分，名為*按嚴重程度排列的未解決警報*和*最早未解決的Keystone警報*。欲了解更多信息，請參閱 link:https://docs.netapp.com/us-en/keystone-staas/integrations/keystone-console.html["NetApp Console中的Keystone儀表板"]。
